# Lab 3.1: Component with State — Advanced Counter

## Обзор
Данная лабораторная работа посвящена глубокому изучению механизмов управления состоянием (`useState`) и передачи данных через пропсы (`props`) в React. В рамках задания реализован продвинутый счетчик шагов с отслеживанием истории и статистики.

## Цель работы
Научиться управлять несколькими переменными состояния внутри одного компонента, обеспечивать независимость экземпляров компонентов и грамотно использовать пропсы для конфигурирования поведения UI.

## Структура проекта в src:
*   `StepCounter.tsx` — Основной компонент счетчика. Реализует логику инкремента/декремента с настраиваемым шагом, хранение истории последних 5 изменений и подсчет общего количества операций.
*   `CounterApp.tsx` — Родительский компонент, демонстрирующий композицию. Рендерит два независимых экземпляра `StepCounter` с разными начальными значениями.
*   `App.tsx` — Точка входа, в которой отображается `CounterApp`.

## Реализованный функционал
1.  **Управление пропсами:** Компонент принимает `initialValue` и `step`, используя их для инициализации состояния без хардкода.
2.  **Множественное состояние:** Отдельные стейты для текущего числа, массива истории и счетчика операций.
3.  **Логика Reset:** Полный сброс всех показателей до начальных значений.
4.  **Независимость:** Каждый экземпляр счетчика в `CounterApp` имеет собственное изолированное состояние.

## Методология
В коде добавлены детальные комментарии, объясняющие взаимодействие пропсов и стейта. Основной акцент сделан на "чистоте" обновления массивов в состоянии (иммутабельность при обновлении истории).

**Инструкция по запуску:**
1. Установите зависимости: `pnpm install`
2. Запустите проект: `pnpm dev`
