# Lab 3.2: Data Loading with useEffect

## Обзор
Работа сфокусирована на управлении побочными эффектами (side effects) в React с помощью хука `useEffect`. Реализован компонент профиля пользователя, загружающий данные из внешнего API (JSONPlaceholder).

## Цель работы
Освоить жизненный цикл функционального компонента, научиться обрабатывать состояния загрузки и ошибок, а также предотвращать утечки памяти через механизм очистки (cleanup).

## Ключевые особенности реализации:
*   **Data Fetching:** Использование `fetch API` внутри `useEffect` для получения данных при монтировании компонента и изменении зависимостей.
*   **AbortController:** Реализована логика отмены сетевого запроса, если компонент размонтируется или ID пользователя изменится до завершения загрузки.
*   **Dependency Array:** Синхронизация загрузки данных с пропсом `userId`. При нажатии на кнопки переключения ID, эффект срабатывает повторно.
*   **Состояния UI:** Четкое разделение интерфейса на состояния "Загрузка...", "Ошибка" и "Данные загружены".

## Структура компонентов:
*   `UserProfile.tsx` — Отвечает за логику запроса и отображение данных конкретного пользователя.
*   `UserDashboard.tsx` (Parent) — Управляет состоянием `userId` и предоставляет интерфейс для переключения между пользователями.
*   `App.tsx` — Корневой компонент для отображения дашборда.

## Технический комментарий
В коде реализован `cleanup function`, который вызывает `abortController.abort()`. Это гарантирует отсутствие ошибок типа "update on unmounted component" и оптимизирует использование сетевых ресурсов.

**Инструкция по запуску:**
1. Установите зависимости: `pnpm install`
2. Запустите сервер: `pnpm dev`
