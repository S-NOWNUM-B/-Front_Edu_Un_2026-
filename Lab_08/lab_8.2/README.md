# Lab 8.2: Next.js - SSR Dashboard и сравнение SSR/SSG

## Обзор
В этой лабораторной реализовано приложение на Next.js с акцентом на серверный рендеринг. Добавлен пользовательский dashboard (SSR), а также страницы для сравнения `getServerSideProps` и `getStaticProps`.

## Цель работы
Научиться применять SSR для данных, которые должны обновляться на каждый запрос, и сравнить его с SSG/ISR на практических примерах.

## Структура проекта
- `pages/index.tsx` - главная страница со списком постов и навигацией.
- `pages/dashboard.tsx` - SSR-страница dashboard через `getServerSideProps`.
- `pages/about.tsx` - SSG-страница с ISR (`revalidate: 60`).
- `pages/about-ssr.tsx` - SSR-страница для сравнения поведения с SSG.
- `pages/posts/[id].tsx` - динамические страницы постов (SSG + ISR).
- `lib/api.ts` - мок-API для постов, авторов, пользователя, уведомлений и аналитики.
- `types/index.ts` - типы `Post` и `Author`.

## Реализованный функционал
1. **SSR Dashboard:** данные пользователя, уведомлений и аналитики подгружаются на каждый запрос.
2. **Параллельная загрузка данных:** `Promise.all` для ускорения получения уведомлений и аналитики.
3. **Обработка ошибок SSR:** fallback-состояние dashboard при сбое загрузки.
4. **SSG + ISR для About:** страница обновляется по таймеру `revalidate: 60`.
5. **SSR vs SSG сравнение:** отдельные страницы с отметкой времени генерации/запроса.
6. **Динамические посты:** `getStaticPaths` + `getStaticProps` с `fallback: "blocking"`.

## SSR vs SSG
- **Скорость первой загрузки:** SSG быстрее за счет предварительной генерации HTML.
- **Актуальность данных:** SSR всегда свежее, так как пересчитывается на каждый запрос.
- **Нагрузка на сервер:** SSR дороже, потому что выполняет вычисления на каждый запрос.
- **Рекомендация в рамках работы:**
  - SSR: `/dashboard`, `/about-ssr`
  - SSG/ISR: `/about`, `/posts/[id]`, главная страница

## Методология
Проект демонстрирует смешанную стратегию рендеринга: статические страницы для предсказуемого контента и SSR для персонализированных/часто меняющихся данных.

**Инструкция по запуску:**
1. Установите зависимости: `pnpm install`
2. Запустите проект: `pnpm dev`
3. Откройте: `http://localhost:3000`
