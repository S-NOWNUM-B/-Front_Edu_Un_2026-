# Lab 8.1: Next.js Pages Router - SSG, ISR и динамические маршруты

## Обзор
Данная лабораторная работа посвящена изучению статической генерации страниц в Next.js (Pages Router). Реализован блог с главной страницей, страницей "О проекте" и динамическими страницами постов.

## Цель работы
Научиться использовать `getStaticProps`, `getStaticPaths` и ISR (`revalidate`) для предварительной генерации контента, а также обработку `notFound` для несуществующих страниц.

## Структура проекта
- `pages/index.tsx` - главная страница блога со списком постов (SSG + ISR).
- `pages/about.tsx` - статическая страница с меткой времени генерации (SSG).
- `pages/posts/[id].tsx` - динамическая страница поста (SSG через `getStaticPaths` + `getStaticProps`).
- `lib/api.ts` - мок-данные постов и авторов, функции доступа к данным.
- `types/index.ts` - TypeScript-интерфейсы `Post` и `Author`.

## Реализованный функционал
1. **SSG для главной страницы:** список постов генерируется заранее через `getStaticProps`.
2. **ISR:** автоматическое обновление статических страниц с `revalidate: 60`.
3. **Динамические маршруты:** генерация путей для `/posts/[id]` через `getStaticPaths`.
4. **Fallback blocking:** новые или отсутствующие пути обрабатываются через `fallback: "blocking"`.
5. **Обработка ошибок данных:** возврат `notFound: true`, если пост или автор не найден.

## Ключевые концепции
- **`getStaticProps`** - получение данных на этапе сборки.
- **`getStaticPaths`** - список динамических маршрутов для предварительной генерации.
- **ISR** - обновление статических страниц без полной пересборки проекта.
- **Pages Router** - файловая маршрутизация через папку `pages`.

## Методология
Проект построен по модели "данные -> статический рендер -> периодическая ревалидация". Это подходит для контента, который не меняется каждую секунду, но должен регулярно обновляться.

**Инструкция по запуску:**
1. Установите зависимости: `pnpm install`
2. Запустите проект: `pnpm dev`
3. Откройте: `http://localhost:3000`
