4. Оптимизацию нужно подтверждать замерами, а не предположениями
5. Разделение UI на небольшие компоненты упрощает контроль ререндеров

### Промпт 4:
"У меня ошибки в коде, помоги их найти и решить"

Что получил: Подсказки по типам для `react-window`, корректному `itemCount` и обработке фильтра.

## 3. Модификация и проверка
- Реализован виртуализированный список на `react-window` (`FixedSizeList`)
- Добавлена фильтрация 10 000 элементов по title/category
- Добавлен обычный список для сравнения подходов
- Создан генератор данных `generateItems` и интерфейс `Item`
- Подключены типы для `react-window` в проекте TypeScript

## 4. Что узнал
1. Виртуализация снижает нагрузку на DOM при больших массивах данных
2. Для быстрого UI важен рендер только видимой области
3. useMemo полезен для фильтрации и кеширования массивов
4. Обычный список быстро деградирует при росте количества элементов
5. Сравнение двух подходов помогает обосновать архитектурные решения
