# Lab 9.1: React Performance - memo, useMemo и useCallback

## Обзор
В лабораторной реализован учебный dashboard для анализа перерендеров компонентов и оптимизации производительности в React.

## Цель работы
Научиться снижать количество лишних ререндеров с помощью `memo`, `useMemo` и `useCallback`, а также отслеживать эффект оптимизаций на практике.

## Структура проекта
- `src/App.tsx` - корневой компонент приложения.
- `src/components/Dashboard.tsx` - главный экран со счетчиком и компоновкой блоков.
- `src/components/Button.tsx` - memo-компонент кнопки инкремента.
- `src/components/UserCard.tsx` - memo-компонент карточки пользователя.
- `src/components/AnalyticsChart.tsx` - блок с тяжелым вычислением, кешируемым через `useMemo`.

## Реализованный функционал
1. **Управление состоянием:** счетчик в `Dashboard` обновляется по кнопке.
2. **Оптимизация компонентов:** `Button` и `UserCard` обернуты в `memo`.
3. **Стабильные колбэки:** `handleIncrement` создан через `useCallback`.
4. **Тяжелые вычисления:** аналитика рассчитывается один раз и кешируется через `useMemo`.
5. **Анализ ререндеров:** в компонентах оставлены `console.log` для визуального контроля поведения.

## Ключевые концепции
- **`memo`** - предотвращает повторный рендер при неизменных props.
- **`useCallback`** - стабилизирует ссылку на функцию между рендерами.
- **`useMemo`** - кеширует результат вычисления до изменения зависимостей.
- **Локализация состояния** - помогает изолировать влияние обновлений.

## Методология
В проекте используется пошаговая оптимизация: сначала выделяются потенциально «дорогие» места (heavy compute и лишние ререндеры), затем применяется мемоизация и проверяется фактический результат через логирование.

**Инструкция по запуску:**
1. Установите зависимости: `pnpm install`
2. Запустите проект: `pnpm dev`
3. Откройте адрес из консоли Vite (обычно `http://localhost:5173`)
