# Lab 9.2: React Performance - виртуализация списков с react-window

## Обзор
Лабораторная посвящена отображению больших наборов данных в React. Реализованы два подхода: виртуализированный список на `react-window` и обычный список для сравнения.

## Цель работы
Понять, как виртуализация влияет на производительность интерфейса при большом количестве элементов, и сравнить ее с классическим рендером массива.

## Структура проекта
- `src/App.tsx` - вывод обоих вариантов списков на одной странице.
- `src/components/VirtualList.tsx` - виртуализированный список на `FixedSizeList`.
- `src/components/RegularList.tsx` - обычный список (без виртуализации) для сравнения.
- `src/utils/generateItems.ts` - генерация тестовых данных и тип `Item`.
- `src/types/react-window.d.ts` - типы для библиотеки `react-window`.

## Реализованный функционал
1. **Виртуализация:** отображение 10 000 элементов с рендером только видимой области.
2. **Фильтрация:** поиск по `title` и `category` в виртуальном списке.
3. **Сравнительный стенд:** отдельный блок обычного списка на 1 000 элементов.
4. **Оптимизация вычислений:** массивы и фильтрация завернуты в `useMemo`.
5. **Гибкая генерация данных:** единый генератор элементов для обоих сценариев.

## Ключевые концепции
- **Virtualization** - уменьшение DOM-нагрузки за счет рендера видимых строк.
- **`FixedSizeList`** - компонент `react-window` для списков с фиксированной высотой строки.
- **`useMemo`** - предотвращает лишние пересчеты при фильтрации.
- **Сравнительный анализ** - позволяет наглядно увидеть разницу подходов.

## Методология
Приложение построено как демонстрационный стенд: сначала показывается оптимизированный сценарий с виртуализацией, затем обычный сценарий. Это упрощает визуальную оценку скорости скролла и реакции интерфейса.

**Инструкция по запуску:**

2. Запустите проект: `pnpm dev`
