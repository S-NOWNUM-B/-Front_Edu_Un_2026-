## 2. Конкретные промпты

### Промпт 1:
"У меня не работает проект. Мобильное приложение последней версии"

Что получил: AI проанализировал конфигурацию проекта (`package.json`) и помог выявить конфликт версий (canary-сборку Expo). Был понижен до стабильного SDK 52, затем переустановлены зависимости и очищен кэш через `npm install --force`.

### Промпт 2:
"Project is incompatible with this version of Expo Go. The installed version of Expo Go is for SDK 54.0.0."

Что получил: AI выполнил обновление проекта до SDK 54.0.0 (совместимого со свежим клиентом Expo Go из App Store), разрешил конфликты peer-зависимостей (`react`, `react-native`) через `npm install --legacy-peer-deps` и `npx expo install --fix`. Успешный запуск проекта.

## 3. Модификация и проверка
- Исправлены нативные проблемы с версиями Expo SDK в проектах (откат с `canary`, обновление до `54.0.0`).
- Произведена полная очистка `node_modules` и `package-lock.json` с последующей переоценкой зависимостей.
- Созданы переиспользуемые UI-компоненты со стилями.

## 4. Что узнал
1. Как правильно работать с версиями Expo SDK и клиентом Expo Go, разрешая ошибки несовместимости версий (SDK mismatch).
2. Как устранять конфликты "peer dependencies" при установке React Native модулей.
3. Создание базовых UI-компонентов мобильного приложения со стилизацией через Flexbox и `StyleSheet` (на примере `Lab 11.1`).
4. Настройка стековой навигации (`@react-navigation/native-stack`) между экранами мобильного приложения.

## 5. Процент использования
