- Дата: 04.04.2026

## 2. Конкретные промпты

### Промпт 1:
"Как правильно тестировать React-компоненты с Testing Library"

Что получил: Объяснение подхода тестирования через поведение (behavior-driven), использование `data-testid` вместо CSS-селекторов, рекомендации по `userEvent` вместо `fireEvent`.

### Промпт 2:
"Как тестировать добавление элемента в список по Enter"

Что получил: Пример с `userEvent.type(input, 'text{Enter}')` и проверкой `onKeyDown` обработчика.

### Промпт 3:
"Как правильно писать describe-блоки для тестов компонентов"

Что получил: Рекомендации по группировке тестов: Rendering → Actions → Edge cases.

### Промпт 4:
"У меня ошибки в коде тестов, помоги их найти и решить"

Что получил: Подсказки по типам `@testing-library/user-event`, корректному `userEvent.setup()` и асинхронным проверкам.

## 3. Модификация и проверка
- Создан компонент TodoList с CRUD-операциями
- Написаны тесты на рендеринг (пустой список, initialTodos)
- Добавлены тесты на добавление (кнопка + Enter)
- Реализованы тесты на toggle и delete
- Добавлен тест на счётчик задач
- Все тесты прошли успешно через `pnpm test`

## 4. Что узнал
1. Testing Library рекомендует тестировать через доступные пользователю элементы
2. `userEvent.setup()` предпочтительнее `fireEvent` для реалистичной эмуляции
3. `data-testid` даёт стабильные селекторы, не зависящие от CSS
4. Группировка `describe` по категориям упрощает чтение отчётов
5. Асинхронные тесты требуют `async/await` с `userEvent`

### Промпт 2:

### Промпт 3:
"Как проверить что элемент удалён из DOM после действия"

Что получил: Разницу между `queryByText` (возвращает null) и `getByText` (бросает ошибку), рекомендацию использовать `expect(...).not.toBeInTheDocument()`.

"У меня ошибки в коде тестов, помоги их найти и решить"

4. Vercel-деплой совместим с Jest-тестами
