# Lab 4.2: Lifting State Up & Article Manager

## Обзор
Работа демонстрирует паттерн "подъема состояния" (lifting state up) в React — ключевую концепцию для управления общим состоянием между несколькими компонентами. Реализован менеджер статей с возможностью добавления, удаления и интерактивного просмотра.

## Цель работы
Освоить технику вынесения состояния в родительский компонент для обеспечения синхронизации данных между дочерними компонентами. Научиться проектировать архитектуру приложения с четким разделением ответственности.

## Архитектура компонентов:
*   `types.ts` — Определение интерфейса `Article` с полями `id`, `title` и `summary`. Обеспечивает строгую типизацию во всем приложении.
*   `ArticleManager.tsx` — **Контейнерный компонент** (parent). Управляет массивом статей (`articles`), состоянием полей ввода (`title`, `summary`) и содержит всю бизнес-логику (добавление/удаление статей).
*   `AddArticle.tsx` — **Презентационный компонент** для добавления новых статей. Получает данные и callback-функции через пропсы, не имеет собственного состояния для полей формы.
*   `ArticleList.tsx` — Компонент-список, который рендерит коллекцию статей, передавая каждую в `ArticleItem`.
*   `ArticleItem.tsx` — Атомарный компонент для отображения одной статьи. Имеет локальное состояние `isOpened` для раскрытия/сворачивания описания статьи.

## Ключевые концепции
1.  **Lifting State Up:** Состояние форм (`title`, `summary`) и список статей хранятся в родительском `ArticleManager`, а не в дочерних компонентах. Это позволяет `AddArticle` и `ArticleList` синхронизироваться через общий источник данных.
2.  **Callback Props:** Функции `onAdd`, `onRemove`, `onChangeTitle`, `onChangeSummary` передаются вниз по дереву компонентов, обеспечивая двустороннюю коммуникацию (child-to-parent).
3.  **Immutable Updates:** При добавлении/удалении статей создается новый массив (`[...articles, newArt]`, `articles.filter(...)`), что соответствует принципам иммутабельности в React.
4.  **Local vs Lifted State:** Состояние `isOpened` остается внутри `ArticleItem`, так как оно не нужно другим компонентам — демонстрация разумного применения паттерна.

## Реализованный функционал
*   Добавление статей с автогенерацией уникального `id` через `Date.now()`
*   Удаление статей по клику на кнопку "Delete"
*   Интерактивное раскрытие/сворачивание описания статьи (toggle)
*   Очистка полей формы после успешного добавления

## Методология
Проект построен по принципу "умные и глупые компоненты" (smart & dumb components). Логика сосредоточена в `ArticleManager`, а презентационные компоненты остаются переиспользуемыми и легко тестируемыми.

**Инструкция по запуску:**

2. Запустите сервер: `pnpm dev`
